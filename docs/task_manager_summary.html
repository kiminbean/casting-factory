Task Manager — 한눈에 보는 요약

큰 주문을 작은 실행 단위로 나누는 작업 분해 엔진 · 2026-04-08
한 문장 요약: 공장 관리자가 승인한 work_order(예: A제품 10개 생산)를 로봇이 바로 실행할 수 있는 작은 Task들(주형 제작, 이송, 탈형…)로 쪼개고, 같은 item의 Task는 정해진 순서를 반드시 지키면서 Task Allocator에게 넘겨주는 역할을 합니다.

① 주문 계층 구조

주문이 실제 로봇 명령이 되기까지 4단계를 거칩니다:

customer_order ← 사무실에서 고객 주문 검토·승인 │ ▼ work_order ← 공장 관리자가 "생산 시작" 버튼 클릭 예) A제품 10개 생산 주문 │ ▼ item_id ← work_order의 개별 생산 단위 (1개 = 1개 item) 예) A제품 10개 → item_001 ~ item_010 (각각 독립 실행) │ ▼ Task ← 로봇이 바로 실행할 수 있는 최소 명령 단위 예) item_001의 Task 순서: 주형 제작 → 이송 → 탈형 → 적재 …
핵심 규칙: 같은 item_id 안의 Task들은 반드시 정해진 순서대로 실행해야 합니다. 하지만 서로 다른 item_id(예: item_001과 item_002)는 동시에 진행해도 됩니다.

② 핵심 개념 3가지

✂️

1. 작업 분해 (Decomposition)

큰 work_order를 item_id 단위로 나누고, 각 item_id마다 실행 순서가 있는 Task 목록을 생성합니다.

📋

2. 순서 보장 (Sequencing)

같은 item_id의 Task는 앞 Task가 완료된 후에만 다음 Task가 "준비(Ready)" 상태가 됩니다. 선행 조건 없으면 즉시 실행 가능.

📤

3. 할당 요청 전달

Ready 상태가 된 Task를 Task Allocator의 Ready Task Pool에 넣어줍니다. 누가 실행할지는 Allocator가 결정합니다.

③ Task 실행 순서 (Confluence 정의 기준)

item_id 하나를 처리하기 위해 아래 순서를 반드시 지켜야 합니다:

🔥 생산 Task (A제품 1개 만들기)
  1. 주형 제작 (로봇팔) + 주조 구역 이동 (AMR, 병렬 가능)
  2. 탈형 & 상차 (로봇팔)
  3. 후처리 구역 이동 (AMR)
  4. 자동 수거 위치 이동 (AMR)
  5. 적재 구역 이동 (AMR)
📦 적재 Task (AMR이 적재구역 도착 상태에서 시작)
  1. 하차 (로봇팔)
  2. 적재 (로봇팔) + 대기구역으로 이동 (AMR, 병렬 가능)
🚛 출고 Task
  1. 적재 구역 이동 (AMR)
  2. 출고 & 상차 (로봇팔)
  3. 출고 구역으로 이동 (AMR)

④ 입력과 출력

📥 입력 (받는 것)

work_order공장 관리자가 승인한 생산 지시
task 완료 신호Execution Monitor로부터 "Task A 완료" 이벤트
task 실패 신호재시도 또는 오류 처리 필요 알림

📤 출력 (내보내는 것)

Ready Task지금 당장 실행 가능한 Task → Task Allocator에 전달
상태 업데이트Task·work_order 진행 상황 → DB 반영, UI 갱신
완료 신호모든 Task 완료 시 work_order를 "완료"로 전환

⑥ 구현 순서

  1. work_order 감시: 공장 관리자가 "생산 시작" 누르면 작업 대기열에 등록
  2. item_id 생성: work_order의 수량(10개)만큼 item_id 분할 (item_001~010)
  3. Task DAG 생성: 각 item_id에 대해 실행 순서가 있는 Task 목록 생성
  4. Ready Task 발행: 선행 조건 없는 Task를 즉시 Ready Task Pool에 삽입
  5. 완료 이벤트 수신: Task 완료 시 다음 Task를 Ready Pool에 추가
  6. work_order 완료 판정: 모든 item의 모든 Task 완료 시 work_order 종료
미결 사항 / 주의:
  • Task 실패 재시도 정책 — 로봇 이상/센서 실패 시 몇 번 재시도할지 정해야 함
  • 후처리 Task — 사람이 수동 처리 후 "완료" 버튼을 눌러야 다음 Task가 진행됨
  • item_id 수 조정 — A제품 10개 중 일부 실패 시 나머지 item 처리 방침 결정 필요

⑦ 쉬운 용어 사전

customer_order고객이 사무실에 넣은 주문. 사무실 직원이 검토·승인
work_order공장 관리자가 "생산 시작"을 누른 승인된 주문 (예: A제품 10개)
item_idwork_order를 개별 단위로 쪼갠 것 (A제품 1개 = 1개 item_id)
Task로봇이 바로 실행할 수 있는 최소 명령 단위 (주형 제작, 이송, 탈형 등)
Ready Task선행 Task가 완료되어 지금 당장 실행 가능한 Task
선행 조건이 Task 시작 전에 반드시 완료되어야 하는 Task (예: 탈형은 주조 후에만 가능)
