③ 구현 계획 — 미구현 TODO 2건

3.1 B12 — 비상 정지 (Emergency Stop)

현황 (production.py:424-433): QMessageBox 경고만 출력하고 실제 장비에는 명령이 전달되지 않는다.

요구 사항

  • 클릭 즉시(≤ 500 ms) 공정상 모든 로봇(Conveyor · AMR · Cobot)에 STOP 명령
  • FMS 자동 진행 시퀀서(Mgmt FMS_AUTOPLAY) 일시 정지
  • 이미 할당된 운송 작업(TransportTask)은 "paused" 상태로 동결
  • DB alerts 에 type="emergency_stop" severity="critical" 로 INSERT
  • 사용자 확인(QMessageBox) 후 한 번 더 확인 요구 (double-confirm)
  • 해제(Resume)는 별도 "재개" 액션으로 분리 (본 SPEC 외)

구현 단계

Step 1. proto 확장 (backend/management/proto/management.proto) 
message EmergencyStopRequest {
  string reason   = 1;   // "operator_button" | "sensor_fault" | ...
  string issuer   = 2;   // "pyqt_factory_op" 등
}
message EmergencyStopResponse {
  bool   accepted          = 1;
  int32  stopped_robots    = 2;   // 실제 STOP 수신 로봇 수
  string alert_id          = 3;
  string reason            = 4;
}
service ManagementService {
  // ... 기존 ...
  rpc EmergencyStop(EmergencyStopRequest) returns (EmergencyStopResponse);
}
Step 2. Management 핸들러 (backend/management/server.py)
  • self.task_manager에 pause_autoplay() 호출 → FMS_AUTOPLAY daemon thread 에 정지 플래그
  • active 로봇 일괄 조회 → robot_executor.execute_command(robot_id, "STOP") fan-out
  • alerts 테이블 INSERT (models_mgmt.Alert)
  • 응답에 실제 STOP 전달된 로봇 수와 alert_id 반환
Step 3. PyQt ManagementClient 메서드 (monitoring/app/management_client.py) 
def emergency_stop(self, reason: str = "operator_button") -> EmergencyStopResponse:
    self._ensure_channel()
    req = management_pb2.EmergencyStopRequest(
        reason=reason, issuer=os.environ.get("USER", "pyqt")
    )
    return self._stub.EmergencyStop(req, timeout=1.5)
Step 4. PyQt 핸들러 (production.py:424) 
def _on_emergency_stop(self) -> None:
    confirm = QMessageBox.warning(self, "비상 정지 재확인",
        "모든 장비를 즉시 정지합니까? 되돌리려면 별도 재개 작업이 필요합니다.",
        QMessageBox.Yes | QMessageBox.No, QMessageBox.No)
    if confirm != QMessageBox.Yes: return

    try:
        resp = self._mgmt.emergency_stop(reason="operator_button")
    except grpc.RpcError as exc:
        QMessageBox.critical(self, "비상 정지 실패",
            f"Mgmt gRPC 오류: {exc.code()}\n물리 E-Stop 버튼을 누르세요.")
        return
    QMessageBox.information(self, "비상 정지 완료",
        f"로봇 {resp.stopped_robots}대 STOP · alert {resp.alert_id} 생성")
    self._control_panel.set_estop_active(True)  # UI lock
Step 5. ControlPanel UI 잠금
  • ControlPanel.set_estop_active(True): 빨간 border + AUTO/MANUAL 토글 비활성
  • 재개(Resume) 전까지 _on_mode_changed, _on_emergency_stop 다시 누를 수 없도록

3.2 B13 — AUTO/MANUAL 모드 토글

현황 (production.py:435-437): mode = "AUTO" if auto else "MANUAL" 문자열만 계산하고 버림.

요구 사항

  • AUTO 모드: FMS_AUTOPLAY 시퀀서가 item FSM 자동 진행
  • MANUAL 모드: 시퀀서 정지, 모든 상태 전이는 PyQt 버튼(B14) 또는 현장 HW 트리거로만
  • 모드 전환이 성공하면 UI 상단 뱃지(AUTO/MANUAL) 업데이트
  • 모드는 Mgmt 프로세스 상태이므로 재시작 시 기본값은 FMS_AUTOPLAY 환경변수

구현 단계

Step 1. proto 확장 
message SetAutoModeRequest  { bool auto = 1; }
message SetAutoModeResponse { bool accepted = 1; bool current_auto = 2; }
rpc SetAutoMode(SetAutoModeRequest) returns (SetAutoModeResponse);
Step 2. Management 핸들러
  • fms_sequencer.set_autoplay(auto) 호출로 시퀀서 pause/resume
  • 응답의 current_auto 로 실제 적용 상태 반환 (race condition 대비)
Step 3. PyQt 핸들러 
def _on_mode_changed(self, auto: bool) -> None:
    try:
        resp = self._mgmt.set_auto_mode(auto)
    except grpc.RpcError as exc:
        QMessageBox.warning(self, "모드 전환 실패",
            f"Mgmt gRPC 오류: {exc.code()}")
        self._control_panel.mode_toggle.setChecked(not auto)  # 원복
        return
    self._status_bar.showMessage(
        f"모드 전환: {'AUTO' if resp.current_auto else 'MANUAL'}", 3000)
