4. gRPC keep-alive 전 구간 적용

ff3d796

PyQt 카메라 뷰 "멈춤" 사건 재발 방지

문제 배경

현상: GetLatestFrame 제거 커밋 후 서버 재기동 직후, 기존 PyQt 인스턴스의 카메라 뷰가 멈춤.
원인: 서버 재기동으로 TCP 가 단절됐지만 PyQt 측 worker 의 자동 재연결이 오래 걸림 (OS TCP timeout 의존). stale connection 을 빠르게 감지할 수단 없음.

적용 옵션

options = [
  ("grpc.keepalive_time_ms",          30000),   # 30s 마다 HTTP/2 PING
  ("grpc.keepalive_timeout_ms",       10000),   # 10s 내 PONG 없으면 단절로 판정
  ("grpc.keepalive_permit_without_calls", 1),   # idle stream 에서도 ping 허용
  ("grpc.http2.max_pings_without_data",   0),   # 무제한 허용 (서버 거부 방지)
  # 서버 전용 추가:
  ("grpc.http2.min_ping_interval_without_data_ms", 20000),  # 클라 ping 수용 기준
]

적용 지점 (3 곳)

  • backend/management/server.py — grpc.server(..., options=[...]) + ThreadPool max_workers 10 → 32 (스트림 워커 점유 여유)
  • monitoring/app/management_client.py — insecure/mTLS 채널 모두
  • jetson_publisher/publisher.py — insecure/mTLS 채널 모두

효과

  • 서버 재기동·Tailscale 단절 시 최대 40초 내 예외 발생 → worker 재연결 트리거
  • 이전엔 OS TCP 타임아웃(~수분) 까지 silent hang

펌웨어 v5.0 (v4.0 기반 재작성)

6034604 → bee8880

firmware/conveyor_v5_serial/

첫 커밋(6034604)은 스켈레톤 — v4.0 의 성숙한 센서 로직(anti-crosstalk, debounce, MIN_RUN_MS)을 놓침. 사용자 피드백 후 bee8880 에서 v4.0 기반으로 재작성.

v4.0 에서 유지

  • TOF250 ASCII 파서 + 500ms debounce
  • Anti-crosstalk — TOF1 감지 중엔 TOF2 raw 무시 (IR 940nm 교란 방지)
  • MIN_RUN_MS 1초 motor start 후 TOF2 false trigger 차단
  • 5-state: IDLE/RUNNING/STOPPED/POST_RUN/CLEARING
  • CLEARING timeout 5s 안전장치
  • sim_entry / sim_exit / reset / status 명령

v4.0 → v5.0 변경

  • WiFi/MQTT 기본 비활성 (#define ENABLE_WIFI_MQTT 0) — 1 로 바꾸면 v4.0 동작 복원
  • ST_STOPPED 5초 자동 탈출 제거 — Jetson RUN 수신 시에만 POST_RUN
  • POST_RUN 4s (사용자 요구)
  • Serial 간단 토큰 라인 추가: BOOT / STATE:<NAME> / STOPPED / STARTED / DONE / PONG / ERR
  • 명령 alias 확장: RUN / camera_ok / inspect_done / PING / STATUS (대/소문자 모두)

Jetson EspBridge (publisher 내 스레드)

e2beee5

jetson_publisher/esp_bridge.py

  • QThread 수준의 독립 스레드 — publisher.py 에 opt-in (ESP_BRIDGE_ENABLED=1)
  • 환경변수: ESP_BRIDGE_PORT / BAUD / AUTO_RUN / INSPECTION_DELAY_MS
  • 핵심 로직:

    • Serial 끊기면 10s backoff 재연결
  • 단위 테스트 6/6 통과 (tests/test_esp_bridge.py, mock serial)
