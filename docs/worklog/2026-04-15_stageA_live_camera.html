2026-04-15 · Stage A · PyQt 품질 페이지 라이브 카메라 뷰 동작 확인

Jetson Orin NX 에 연결된 USB 카메라(Logitech C920) 영상을 PyQt 모니터링 앱의 품질 검사 페이지 카메라 뷰에 1Hz 로 실시간 표시한다. 기존 시뮬레이션 위젯을 실 프레임 렌더링으로 승격하면서 PASS/FAIL 오버레이 구조는 그대로 유지해 Stage B(실제 추론) 도입 시 자연스럽게 이어지도록 설계했다.

gRPC RPC 1 추가PyQt 위젯 확장1Hz polling판정은 Stage B 대기

1. 배경과 목표

V6 이미지 파이프라인 상태 점검 중, 사용자가 "카메라 뷰에 실제 영상이 보이는가? PASS/FAIL 로 나오는가?" 를 물었다. 조사 결과 현재 구성은 다음과 같았다.

Before (2026-04-15 오후 전):
· 카메라 뷰는 CameraLiveView 시뮬레이션 위젯 — QPainter 로 그라데이션+스캔라인 렌더
· PASS/FAIL 판정 데이터는 레거시 FastAPI /api/quality/vision-feed 에서 mock 수신
· Jetson → Mgmt image_sink 로 쌓이는 실제 프레임은 PyQt 와 전혀 연결되지 않음

두 단계로 분해했다:

  • Stage A (지금 진행) — 실제 프레임 스트림을 PyQt 카메라 뷰에 표시. 추론은 아직 없음.
  • Stage B (AI Server 모델 확정 후) — IP snapshot 에 대한 PASS/FAIL 추론 + DB/스트림 노출.

2. 설계 결정

항목선택근거
전송 방식서버 polling (GetLatestFrame unary)구현 단순, 테스트 용이. 추후 server streaming 으로 승격 여지
주기1Hz (MGMT_LIVE_FRAME_POLL_MS=1000)Jetson 2Hz 생산 속도의 절반. 대역·CPU 부담 낮음
sequence 기반 dedup클라이언트 쪽에서 비교같은 seq 이미지로 paintEvent 중복 트리거 방지
기존 시뮬레이션 위젯 처리pixmap 이 있을 때만 실 프레임 표시, 없으면 기존 렌더 유지연결 끊기면 STANDBY 표시로 자연스러운 fallback
PASS/FAIL 오버레이그대로 유지 (mock 소스)Stage B 에서 소스만 교체하면 됨

4. 구현 상세

4.1 proto 정의 — 신규 RPC

// backend/management/proto/management.proto
service ManagementService {
  ...
  rpc Health(Empty) returns (Empty);
  rpc GetLatestFrame(LatestFrameRequest) returns (LatestFrameResponse);
}

message LatestFrameRequest {
  string camera_id = 1;
}
message LatestFrameResponse {
  bool      available   = 1;
  string    camera_id   = 2;
  string    encoding    = 3;   // "jpeg"
  uint32    width       = 4;
  uint32    height      = 5;
  bytes     data        = 6;
  uint64    sequence    = 7;
  Timestamp captured_at = 8;
  Timestamp received_at = 9;
}

4.2 서버 servicer — image_sink 조회

# backend/management/server.py
def GetLatestFrame(self, request, context):
    cam = request.camera_id or ""
    frame = image_sink.latest(cam) if cam else None
    if not frame:
        return management_pb2.LatestFrameResponse(
            available=False, camera_id=cam,
        )
    return management_pb2.LatestFrameResponse(
        available=True, camera_id=cam,
        encoding=frame.get("encoding", ""),
        width=int(frame.get("width", 0) or 0),
        height=int(frame.get("height", 0) or 0),
        data=frame.get("data", b""),
        sequence=int(frame.get("sequence", 0) or 0),
        captured_at=management_pb2.Timestamp(iso8601=frame.get("captured_at","")),
        received_at=management_pb2.Timestamp(iso8601=frame.get("received_at","")),
    )

4.3 PyQt 클라이언트 헬퍼

# monitoring/app/management_client.py
def get_latest_frame(self, camera_id: str) -> dict | None:
    try:
        resp = self._stub.GetLatestFrame(
            management_pb2.LatestFrameRequest(camera_id=camera_id),
            timeout=2.0,
        )
    except grpc.RpcError as e:
        logger.warning("GetLatestFrame failed: %s", e)
        return None
    if not resp.available:
        return None
    return {
        "camera_id": resp.camera_id, "encoding": resp.encoding,
        "width": int(resp.width), "height": int(resp.height),
        "data": bytes(resp.data), "sequence": int(resp.sequence),
        "captured_at": resp.captured_at.iso8601,
        "received_at": resp.received_at.iso8601,
    }

4.4 CameraLiveView 확장 — 실 프레임 렌더

# monitoring/app/widgets/camera_view.py
def set_frame_bytes(self, data: bytes, encoding="jpeg",
                    sequence=0, received_at=""):
    if not data:
        self._pixmap = None
        self.update(); return
    img = QImage.fromData(data, encoding.upper() or "JPEG")
    self._pixmap = None if img.isNull() else QPixmap.fromImage(img)
    self._frame_sequence = int(sequence)
    self._frame_received_at = received_at or ""
    self.update()

# paintEvent 분기
if self._pixmap is not None and not self._pixmap.isNull():
    scaled = self._pixmap.scaled(w, h, Qt.KeepAspectRatio, Qt.SmoothTransformation)
    painter.drawPixmap((w-scaled.width())//2, (h-scaled.height())//2, scaled)
    self._draw_overlay(painter, w, h)    # 브래킷 + PASS/FAIL 라벨 + 하단 바
    return
# 없으면 기존 시뮬레이션 렌더 (원형 제품 + 스캔라인 + 격자)

오버레이(_draw_overlay) 는 실 프레임/시뮬레이션 모두에서 재사용되어 PASS/FAIL/CHECKING/STANDBY 라벨과 ROI 코너 브래킷이 항상 동일한 위치에 그려진다. 하단 정보 바는 "카메라: LIVE · seq N · 수신 시각" 으로 바뀌어 실시간 피드 여부를 시각적으로 구분한다.

4.5 QualityPage — 1Hz 폴링 타이머

# monitoring/app/pages/quality.py
_LIVE_CAMERA_ID = os.environ.get("MGMT_IP_CAMERA_ID", "CAM-INSP-01")
_FRAME_POLL_MS  = int(os.environ.get("MGMT_LIVE_FRAME_POLL_MS", "1000"))

# __init__
self._frame_timer = QTimer(self)
self._frame_timer.setInterval(_FRAME_POLL_MS)
self._frame_timer.timeout.connect(self._poll_live_frame)
self._frame_timer.start()

def _poll_live_frame(self):
    mgmt = self._ensure_mgmt_client()   # lazy ManagementClient
    if mgmt is None: return
    frame = mgmt.get_latest_frame(_LIVE_CAMERA_ID)
    if not frame: return
    seq = int(frame.get("sequence", 0))
    if seq == self._last_frame_seq: return   # dedup
    self._last_frame_seq = seq
    self._camera.set_frame_bytes(
        data=frame["data"], encoding=frame["encoding"],
        sequence=seq, received_at=frame["received_at"],
    )

8. 알려진 한계와 Stage B 로드맵

현재 한계:
  • PASS/FAIL 은 실 추론 아님 — FastAPI mock 데이터 기반
  • 폴링 방식 — 서버 파이프는 pub/sub 이 아님. client streaming 으로 바꾸면 대기 시간 ↓
  • 다중 카메라: 현재 1개 카메라 ID 만 하드코딩. 페이지 탭/셀렉터 필요
  • PyQt 는 Mac 에서 Mgmt 서버와 같은 호스트 — 프로덕션 Factory Operator PC 배포 미정

Stage B 예정 작업 (AI Server 모델 확정 후)

  1. AI Server 추론 gRPC 엔드포인트 정의 (입력: ImageFrame, 출력: VerdictResponse)
  2. image_forwarder 가 IP snapshot 업로드 후 추론 호출 → 결과 DB 기록
  3. PyQt WatchInspections stream 또는 기존 WatchItems 에 verdict 채널 추가
  4. CameraLiveView.set_result() 의 소스를 mock 에서 실 verdict 로 교체
  5. docs/worklog/ Stage B 보고서 추가

9. 회고

사용자의 "실제 영상이 보이는가?" 질문 한 번으로 현재 구성의 gap(시뮬레이션 vs 실제 스트림) 을 명확히 드러낼 수 있었다. Stage A 는 1.5 시간 내 완결 — gRPC RPC 1개 + 위젯 확장 + 1Hz 타이머만으로 충분했다. 이전 커밋에서 시뮬레이션 위젯을 QFrame 기반으로 잘 추상화해 둔 덕에 set_frame_bytes() 한 메서드 추가로 실 프레임 모드로 승격할 수 있었다. Stage B 에서도 오버레이 구조를 건드리지 않고 판정 소스만 교체하면 된다.
