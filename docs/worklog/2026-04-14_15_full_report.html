Waypoint + Dijkstra + Backtrack Yield (340 LOC, 22 tests)

알고리즘 스택

  1. Waypoint 그래프 — 1m × 2m 맵, 8 노드 / 8 양방향 edges
    • HOME, JUNCT_A/B, POST, INSP_IN/OUT, LOAD, SHIP
  2. Dijkstra 최단 경로 — heapq 기반, exclude_edge 옵션
  3. Edge time-window 예약 — 스레드 안전
  4. Backtrack Yield 3단계
    1. 단순 reservation 시도 (priority 비교로 강제 회수 가능)
    2. 차단 edge 회피 우회 경로
    3. 출발 시각 지연 (충돌 종료 후 + safety margin)

priority 규약: 작은 숫자 = 우선. 자기보다 priority 큰(낮은) 기존 예약은 강제 양보.

검증 (22/22 pass)

  • ① HOME→SHIP Dijkstra: 5점 4edges 6.83s
  • ② 2대 동일 경로: 후순위가 7.8s 대기 양보
  • ③ higher priority → 기존 점유 강제 회수
  • ④ 3대 동시 (HOME→SHIP / HOME→LOAD / POST→SHIP): 충돌 0
  • ⑤ JUNCT_A↔JUNCT_B 차단 시 POST 경유 우회
  • ⑥ gRPC PlanRoute RPC E2E
