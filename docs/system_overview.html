① 3가지 개념 비교

MES

Manufacturing Execution System
분류: ISA-95 Level 3 표준
역할: 생산 계획~실행~품질~재고 통합
시간: 분/시간/일
사용자: 관리자/오퍼레이터
표준: ISA-95, MESA 11
우리 위치: README "Casting Factory MES"

Open-RMF

Open Robotics Middleware Framework
분류: 오픈소스 ROS 2 미들웨어
역할: 다중 로봇/AGV 통합 조율
시간: 초/분
사용자: 시스템 통합자
제공: rmf_traffic, rmf_task, fleet_adapter
우리 위치: 미사용 (Phase 3 검토)

관제 시스템

Control / Monitoring System (HMI/SCADA)
분류: 일반 용어 (ISA Level 2 SCADA)
역할: 실시간 감시 + 제어
시간: ms/초
사용자: 관제실 운영자
특징: 텔레메트리, 알람, E-Stop
우리 위치: PyQt5 monitoring/ 앱

⑥ 4-Layer 시스템 모델 (Confluence SA v3 기준)

📚 Confluence "System Architecture v3" 에서 정의한 인터페이스 계층 분리 모델
Interface Layer UI 분리 + API 표준화
Interface Service UI(Admin/Customer) 와 시스템 사이 입출력 창구. 표준 JSON API 변환
왜 분리하는가 UI ↔ 백엔드 Decoupling: 웹/앱/태블릿 UI 바뀌어도 backend 그대로 사용 가능
⇣ ⇡
Control Layer (Decision) 중앙 두뇌
Control Service 요청 검증 / 비즈니스 로직 / DB 저장 / AI 호출 / HW 명령 생성
우리 매핑 FastAPI Backend (port 8000) — 우리 시스템의 Control Service
⇣ ⇡
Execution Layer (FMS) 실행 매니저
Fleet Management System "무엇을 할지"가 아니라 "어느 로봇이, 어느 경로로, 언제" 결정
Robot Allocator 로봇 작업 분배 + 충돌 방지 + 경로 계획
우리 매핑 현재 FastAPI scheduler.py + task_allocator (자체 구현, Open-RMF 미사용)
⇣ ⇡
Equipment Layer 실제 장비
Conveyor Belt ESP32 Controller · 시작/정지/상태 보고
Laser (TOF) 제품 유무/높이 감지
Camera 이미지 획득 → AI Server
Robot Arm MyCobot280 · pick/place
AMR Pinkypro · Nav2 자율주행
핵심 설계 원칙 (Confluence v3 피드백 반영):
• Interface Layer 도입으로 UI 변경 시 backend 무영향 → 유지보수 비용↓
• DB와 AI는 Control Service를 반드시 거침 → 시스템 일관성 유지
• UI는 DB에 직접 저장 금지 → DB 구조 변경 시 UI 영향 차단

⑧ 주조 / 물류 도메인 용어 사전

📚 Confluence "Terminology" 페이지 — 통일된 언어 사용을 위한 핵심 용어

🔥 주조 (Casting)

한글 (영문)정의
주조 (Casting)금속을 녹여 주형에 부어 형상을 만드는 공정
용해 (Melting)금속을 고온에서 녹여 액체 상태로 만드는 과정
용탕 (Molten Metal)녹은 금속 상태
주괴 (Ingot)용해로에 투입하여 용탕으로 만들어지는 것
용해로 (Melting Furnace)금속 등을 녹이는 설비. 우리 시스템: FRN-001
도가니 (Crucible)용해로 내부에서 금속을 녹이는 용기 (고정 형태)
래들 (Ladle)용해로 외부에서 용탕을 옮기고 붓는 용기 (이동 형태)
주형 (Mold)금속을 붓는 틀. 사형(모래) / 금형(영구) 구분
주형사 (Molding Sand)모래 + 결합제(벤토나이트) + 수분으로 구성
패턴 (Pattern)주형을 만들기 위한 원형 모델
조형 (Molding)틀을 만드는 행위 (Process)
주탕 (Pouring)용탕을 주형에 붓는 과정 (자동화 핵심 구간)
탕구 / 러너 / 게이트Sprue / Runner / Gate — 용탕 흐름 통로 시스템
압탕 (Riser)응고 시 수축 보정용 용탕 저장부
냉각 / 응고Cooling / Solidification — 용탕이 식어 고체로 변환
탈형 (Shakeout)냉각된 주물에서 주형을 제거하는 과정
주물 (Casting Product)최종 결과물 (우리: 맨홀뚜껑, 그레이팅 등)
후처리 (Finishing)표면 다듬기 — Fettling, Deburring, Shot Blasting

🏷 결함 유형 (Defect)

기공 (Porosity)내부에 기포/구멍 발생
수축 결함 (Shrinkage)응고 시 부피 감소로 발생
콜드 셧 (Cold Shut)두 용탕 흐름이 만났을 때 융합 실패 — 선 모양 이음새
미스런 (Misrun)용탕이 주형을 끝까지 채우지 못함

📦 물류 (Logistics)

물류 (Logistics)제품/자원 흐름 (이동·보관·처리) 효율 관리
입고 (Receiving / Inbound)외부에서 창고로 자재를 들여오는 과정
출고 (Shipping / Outbound)창고에서 고객/다음 공정으로 보내는 과정
이송 (Transport)물류 거점 간 / 공정 간 제품 이동
적치 (Storage / Put-away)제품을 지정 위치에 보관
피킹 (Picking)주문에 맞게 필요 제품 선택/수집
패킹 (Packing)출하 위해 포장
재고 동기화 (Inventory Sync)실제 재고와 WMS 데이터 일치
슬로팅 (Slotting)창고 내 상품 배치 최적화 전략
Cross Docking입고 후 보관 없이 바로 출고로 연결
Golden Zone작업 효율 가장 높은 적치 위치 (허리~가슴 높이)
Scan Lane이동 중 바코드/RFID 재고 인식 구간

🤖 자동화 시스템

WMS (Warehouse Management System)창고 내 재고 및 작업 흐름 관리 시스템
FMS (Fleet Management System)다수 로봇/AGV/AMR 작업 할당 및 경로 제어
AMR (Autonomous Mobile Robot)자율 경로 판단으로 이동하는 물류 로봇 · 우리: Pinkypro
AGV (Automated Guided Vehicle)정해진 경로를 따라 이동하는 자동 운반 장치
로봇 셀 (Robot Cell)로봇 중심 작업 단위 시스템
비전 시스템 (Vision System)카메라 기반 인식/검사 — 우리: Jetson Orin NX + YOLO
택트 타임 (Takt Time)생산 목표 달성을 위한 제품 1개당 필요 시간
