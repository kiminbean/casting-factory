Interface Service 치트시트

V6 canonical · backend/app/ FastAPI :8000 · 2026-04-21 · 판정 양호 (PASS 18/20, 운영 blocker 1)

브라우저(관리자/고객) ─HTTPS─▶ Interface :8000 ─gRPC─▶ Management :50051 ─ROS2/Serial─▶ 설비 └─▶ DB (PG 16 + TimescaleDB, smartcast + public) ◀─┘ Factory Operator PC (PyQt) ───gRPC 직결───▶ Management :50051 (Interface 우회 · SPOF 회피)

계층 경계 5/5 PASS

C-1웹(고객/관리자) ↔ Management 직통신 금지. /api/* HTTP 만.
C-2Interface 는 포트 8000(HTTP) 만 노출. gRPC servicer 등록 금지.
C-3Interface→Mgmt 채널은 clients/management.py 싱글톤 1곳에서만.
C-4Interface 에 ROS2 · MQTT · Serial import 금지 (HW 프로토콜 0).
C-5Management 가 Interface HTTP 역호출 금지. canonical 양방향 화살표는 gRPC full-duplex 의미.

내부 구조 5/6 PASS

I-1gRPC 채널 = lazy + 싱글톤. 첫 호출 시 생성, 프로세스 lifetime 재사용.
I-2Mgmt 장애 → ManagementUnavailable → HTTP 503. Interface 는 안 죽음.
I-3INTERFACE_PROXY_START_PRODUCTION 플래그는 모듈 import 시점 고정. flip 시 worker 전체 재시작.
I-4FastAPI lifespan 이 shutdown 시 ManagementClient.close().
I-5proto stubs 부재 시 graceful degradation (_PROTO_AVAILABLE=False).
I-6mTLS 코드 구현됨 · 현재 MGMT_GRPC_TLS_ENABLED=0 · 운영 전 활성화 필요.

고객 포털 계약 3/4 PASS

O-1발주 수신 단일 엔드포인트 POST /api/orders/customer (원자 트랜잭션).
O-2이메일 조회 GET /api/orders/lookup?email= 없으면 [] (404 아님).
O-3제품/하중 등급 마스터 공개 읽기 전용.
O-4운영 blocker: POST /api/orders/{id}/status 권한 가드 부재. 단기 nginx basic auth, 장기 JWT.

관리자 포털 계약 5/5 PASS

M-1주문 6단계: 접수→승인→생산→생산완료→출고→완료 (OrdStat append).
M-2생산 실시간 뷰 (items · equip_tasks · equip_stats) REST 제공.
M-3품질 · 물류 · 재고 조회 REST 제공 (DB 직결, Mgmt 경유 안 함).
M-4실시간은 소비자 적합형: PyQt=gRPC 직구독, Admin=5초 polling, Customer=on-demand.
M-4a설계 결정: Mgmt→Interface stream relay 현 단계 비도입. 근거 4건 (실시간 소비자는 PyQt 직결·polling DB 부하 trivial·복잡도·gRPC 프로토콜 속성).
M-4b승격 트리거: ① Critical SLA<2s ② Admin 동접≥30 ③ 웹 현장제어 UX. 하나라도 성립 시 Phase C-3.
M-5생산 개시 Mgmt 위임 (SPEC-C2 flag-gated proxy).

데이터 계약 3/3 PASS

D-1Interface + Management 같은 PG + 같은 DATABASE_URL. 분리 금지.
D-2smartcast v2 (27테이블) 우선. public legacy 4개(Alert/HandoffAck/TransportTask/RfidScanLog) 는 models_mgmt.py 에서 선별 import.
D-3SQLite 폴백 없음. DATABASE_URL 미설정 시 fail-fast.

자주 하는 실수 TOP 5

  • C-4 위반: Interface 에서 import rclpy/serial → Mgmt 로 옮길 것
  • C-3 위반: 요청마다 grpc.insecure_channel() 새로 생성 → 싱글톤 ManagementClient.get() 사용
  • I-3 오해: export FLAG=1 했는데 안 먹힘 → worker 전체 재시작 필요
  • D-3 위반: 개발 편의 SQLite → 원격 PG 공유 (Tailscale)
  • D-2 위반: from app.models.models_legacy import * → Item 클래스 충돌. models_mgmt.py 선별 import
