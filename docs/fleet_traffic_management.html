① 서론 — 왜 Traffic Manager인가

공장에 AMR이 2대 이상 동시 운용될 때, Nav2만으로는 다음과 같은 충돌 시나리오를 해결할 수 없습니다:

┌────────────────┐ │ 용해로 영역 │ └───────┬────────┘ │ ┌──────────┼──────────┐ │ │ │ ← 좁은 복도 (1대 통과 가능) │ AMR-1 ↓ AMR-2 │ │ ←── │ ──→ │ │ │ │ └──────────┼──────────┘ │ ┌───────┴────────┐ │ 검사 영역 │ └────────────────┘

Nav2는 각 로봇의 경로만 알 뿐 다른 로봇의 의도를 모릅니다. 그 결과:

  • 복도 중앙 정면 충돌 후 양쪽 모두 교착 (local planner끼리 회피하려다 둘 다 멈춤)
  • 교차점에서 우선순위 없는 deadlock
  • WORKCELL 작업 중인 로봇을 다른 로봇이 밀고 들어옴

Traffic Manager는 이 문제를 "공간에 대한 배타적 접근권(mutex)" 개념으로 해결합니다. 공장을 Zone으로 추상화하고, 로봇은 진입 전 반드시 해당 Zone의 예약(reservation)을 받아야 합니다.

2.3 1m×2m 맵 기준 Zone 설계 감각

  • Zone 한 변은 로봇 대각선(~17cm)의 약 2배 → 30~40cm 권장
  • 맵 전체에 5~15개 Zone이 적정

  • Zone이 너무 크면(>60cm) 병렬성 저하

③ 5가지 예약 전략 스펙트럼

Traffic Manager는 단일 전략이 아닌 여러 전략의 조합입니다. 보수적일수록 안전하지만 공간 활용률이 낮고, 공격적일수록 효율은 높지만 구현 복잡도가 증가합니다.

전략정책장점단점
A. Full Path Lock 경로 전체 예약 후 출발 100% deadlock-free, 단순 공간 활용률 낮음, 긴 대기
B. Sliding Window 권장 앞쪽 N개 Zone만 미리 예약, sliding 활용률 높음, 여전히 안전 중간에 막히면 후퇴 처리 필요
C. Priority Yield 권장 우선순위 낮은 쪽이 회피 교착 해결, 현장 효과↑ 회피 공간 또는 backtrack 필요
D. Negotiation 실시간 협상 프로토콜 최대한 유연 통신 복잡, 검증 어려움
E. Continuous Replanning 시공간 trajectory 예측 (RMF) 이론적 최적 수천 줄, 우리 규모에 과잉
우리 프로젝트 선택: B + C 하이브리드. 기본은 Sliding Window로 sliding 진행, deadlock 감지 시 Priority Yield 발동. E(RMF)는 3대 규모에 과잉, D는 검증 비용이 커서 제외.

④ 전략 B: Sliding Window Lock

4.1 핵심 아이디어

로봇 앞쪽 WINDOW_SIZE개 Zone만 예약하고, 진행하면서 지나온 Zone은 즉시 해제, 앞쪽 Zone은 새로 예약.

초기 상태: [z1][z2][z3][z4][z5][z6][z7][z8] → 목적지 ↑ AMR-1 Window size = 3 → [z1, z2, z3] 예약, 출발 1 tick 후: [✓][z2][z3][z4][z5][z6][z7][z8] ↑ 현재 점유: [z2, z3, z4] ← z1 해제, z4 새로 예약 2 tick 후: [✓][✓][z3][z4][z5][z6][z7][z8] ↑ 현재 점유: [z3, z4, z5]

4.2 동작 순서 (매 tick)

  1. 현재 위치 기준 target_window = path[current_idx : current_idx + WINDOW_SIZE]
  2. to_release = holdings - target_window → 지나간 Zone 즉시 해제
  3. to_acquire = target_window - holdings → 앞쪽 Zone 획득 시도
  4. Release-first 순서: 해제 먼저, 그다음 획득 (자기 자신이 원인이 되어 막히는 현상 방지)
  5. 첫 번째 to_acquire Zone을 못 잡으면 WAITING 전환, 그 외 실패는 window 축소로 허용

4.3 왜 deadlock-free인가

경로 전체가 아니어도 "다음 바로 앞 Zone"만 확보하면 전진 가능합니다. 획득 실패 시 즉시 WAITING으로 빠져서 다른 로봇이 먼저 지나가도록 양보합니다. 획득을 원자적으로 하므로 "부분 점유 → circular wait" 상황이 발생하지 않습니다.

단점: B 단독으로는 정면 대치(head-on) 상황을 해결할 수 없습니다. 두 로봇이 서로 다음 Zone을 기다리며 영원히 대기합니다. → 전략 C가 필요한 이유.

⑤ Deadlock 감지 알고리즘

5.1 감지 조건 (2-robot cycle)

다음 세 조건이 모두 만족되면 두 로봇은 deadlock 상태입니다:

  1. 두 로봇 모두 WAITING 상태
  2. 두 로봇 모두 DEADLOCK_WAIT_THRESHOLD 이상 대기 중
  3. 서로의 next_desired_zone이 상대방의 holdings에 포함됨 (cyclic wait)
def find_deadlock_partner(waiting_robot, all_robots):
    # 너무 짧게 기다렸으면 아직 deadlock 아님
    if datetime.now() - waiting_robot.waiting_since < DEADLOCK_WAIT_THRESHOLD:
        return None

    next_zone = next_desired_zone(waiting_robot)
    blockers = tm.holder_of(next_zone) - {waiting_robot.id}

    for blocker_id in blockers:
        blocker = all_robots[blocker_id]
        if blocker.state != WAITING:
            continue
        blocker_next = next_desired_zone(blocker)
        if blocker_next and waiting_robot.id in tm.holder_of(blocker_next):
            return blocker  # deadlock 확정
    return None

5.3 3-robot cycle 미지원 (의도적 선택)

A→B→C→A의 3각 deadlock은 이론상 가능하지만 3대 AMR 규모에서 실측 빈도가 낮습니다. 2-robot 해결을 먼저 시도하고, 그래도 남은 경우 ERROR 상태로 전환하여 관리자 개입을 요청합니다. 미래 확장 시 waits-for graph의 cycle detection으로 확장 가능.

⑥ 전략 C: Priority Yield

6.1 개념

Deadlock이 감지되면 두 로봇 중 우선순위가 낮은 쪽이 "loser", 높은 쪽이 "winner"가 되어 loser가 회피 동작을 수행합니다.

6.2 우선순위 비교 규칙

  1. 1순위: priority 값 — schedule.py의 7요소 가중 점수
  2. 2순위: FIFO (waiting_since) — 동점이면 먼저 대기 시작한 쪽이 winner
  3. 3순위: ID 사전순 — 완전 동시 진입도 결정론적으로 처리
def pick_loser(a, b):
    # 1순위: priority 비교
    if a.priority != b.priority:
        return (a, b) if a.priority < b.priority else (b, a)

    # 2순위: FIFO — 더 오래 기다린 쪽이 winner
    a_wait = a.waiting_since or datetime.max
    b_wait = b.waiting_since or datetime.max
    if a_wait != b_wait:
        return (a, b) if a_wait > b_wait else (b, a)

    # 3순위: ID 사전순 (tiebreak)
    return (a, b) if a.id < b.id else (b, a)

6.3 배터리 우선순위 부스트

배터리가 임계값(예: 20%) 이하인 로봇은 충전 스테이션 복귀가 급하므로 priority += 1000 부스트를 적용합니다. 이 부스트된 값은 일반 작업 우선순위(최대 100)를 압도하여 항상 winner가 되도록 합니다.

⑦ Pocket 방식 vs Backtrack 방식

loser가 어디로 회피할지 결정하는 두 가지 접근법이 있습니다.

Pocket 방식

공장 맵에 전용 회피 공간(pocket)을 미리 배치. loser는 가장 가까운 pocket으로 이동.

  • ✅ 양방향 정면 대치 해결 가능
  • ✅ 짧은 회피 거리
  • ❌ 공장 설계 시 공간 별도 확보 필요
  • ❌ 1m×2m 맵에서는 공간 부족으로 불가

Backtrack 방식 채택

loser가 자신의 traversal history(지나온 Zone들)를 역순으로 이동하여 winner 경로에서 벗어남.

  • ✅ 추가 공간 불필요
  • ✅ 맵 설계 자유도
  • ✅ 우리 소형 맵에 최적
  • ⚠️ 완전 대치 시 reroute fallback 필요

⑧ Backtrack 상세 알고리즘

8.1 핵심 관찰

로봇이 현재 위치에 있다는 것은 방금 지나온 Zone들이 이미 통행 가능함이 검증되었다는 뜻입니다. 이 이력(traversal history)을 역순으로 탐색하면 별도 pocket 설계 없이 회피 지점을 찾을 수 있습니다.

8.3 Backtrack 탐색 알고리즘

def find_backtrack_target(loser, winner):
    """
    조건:
      1) loser의 traversal_history에 포함
      2) winner의 남은 경로에 포함되지 않음
      3) 현재 capacity 여유 있음
      4) MAX_RETREAT_DISTANCE 이내
    """
    winner_future = set(winner.path)

    retreated = 0
    for past_zone_id in loser.traversal_history:
        retreated += 1
        if retreated > MAX_RETREAT_DISTANCE_ZONES:
            return None

        # 조건 1: winner도 이 Zone을 지나가야 함 → skip
        if past_zone_id in winner_future:
            continue

        # 조건 2: capacity 체크
        past_zone = tm.zones[past_zone_id]
        if not past_zone.can_accept(loser.id):
            continue

        return past_zone_id  # 찾았다

    return None  # 안전한 retreat 지점 없음

8.4 Fallback 전략 4단계

  1. Backtrack 시도 — 가장 가까운 안전 history zone
  2. Reverse Yield — loser/winner 역할을 바꿔 상대방에게 backtrack 시도
  3. Reroute — Nav2 plan_path_avoiding(blocked_zones)로 대안 경로 재계산
  4. Scheduler 재스케줄링 — 작업 자체를 나중으로 미루고 winner 완료 대기
  5. ERROR — 모두 실패 시 알람 + 수동 개입
실패 케이스: 순수 1D 복도(분기 없음)에서 양방향으로 진입한 경우 Backtrack만으로는 해결 불가. 출발 전 예약 검사(dispatch 단계에서 winner 경로가 미점유인지 확인)로 사전 예방하는 것이 근본 대책입니다.

11.2 TrafficManager

class TrafficManager:
    """Zone 점유 상태의 atomic 관리. 단일 스레드 호출 전제."""

    def __init__(self, zones: dict[str, Zone]):
        self.zones = zones

    def try_acquire(self, robot_id: str, zone_id: str) -> bool:
        zone = self.zones[zone_id]
        if not zone.can_accept(robot_id):
            return False
        zone.current_holders.add(robot_id)
        zone.reserved_at[robot_id] = datetime.now()
        return True

    def release(self, robot_id: str, zone_id: str) -> None:
        zone = self.zones.get(zone_id)
        if zone is None:
            return
        zone.current_holders.discard(robot_id)
        zone.reserved_at.pop(robot_id, None)

    def release_all(self, robot_id: str) -> None:
        for zone in self.zones.values():
            if robot_id in zone.current_holders:
                self.release(robot_id, zone.id)

    def holder_of(self, zone_id: str) -> set[str]:
        return set(self.zones[zone_id].current_holders)

    def cleanup_stale(self) -> int:
        """max_hold_sec 초과한 holder 강제 해제."""
        now = datetime.now()
        removed = 0
        for zone in self.zones.values():
            for rid, started in list(zone.reserved_at.items()):
                if now - started > timedelta(seconds=zone.max_hold_sec):
                    self.release(rid, zone.id)
                    removed += 1
        return removed

11.3 SlidingWindowPlanner

class SlidingWindowPlanner:
    WINDOW_SIZE = 2  # 1m×2m 소형 맵 기준

    def __init__(self, traffic_manager: TrafficManager):
        self.tm = traffic_manager

    def update(self, robot: Robot) -> bool:
        """
        Returns:
            True  → 전진 가능
            False → WAITING 전환 필요
        """
        if not robot.path:
            return True

        current_idx = self._current_index(robot)
        target_window = robot.path[current_idx : current_idx + self.WINDOW_SIZE]
        current_holdings = robot.holdings.copy()

        # Step 1: 지나간 Zone 해제 + history 기록
        passed_zones = current_holdings - set(target_window)
        for zid in passed_zones:
            self.tm.release(robot.id, zid)
            robot.holdings.discard(zid)
            robot.traversal_history.appendleft(zid)

        # Step 2: 앞쪽 Zone 새로 획득
        to_acquire = [z for z in target_window if z not in robot.holdings]
        for idx, zid in enumerate(to_acquire):
            if self.tm.try_acquire(robot.id, zid):
                robot.holdings.add(zid)
            else:
                # 첫 앞 Zone조차 못 잡음 → 전진 불가
                if idx == 0:
                    return False
                break  # 나머지는 다음 tick에 재시도 (window 축소)

        return True

    def _current_index(self, robot: Robot) -> int:
        for i, zid in enumerate(robot.path):
            zone = self.tm.zones[zid]
            if zone.polygon.contains(Point(*robot.position)):
                return i
        return 0

11.4 DeadlockDetector

class DeadlockDetector:
    DEADLOCK_WAIT_THRESHOLD = timedelta(seconds=3)  # 소형 맵 기준

    def __init__(self, tm: TrafficManager):
        self.tm = tm

    def find_deadlock_partner(
        self, waiting_robot: Robot, all_robots: dict[str, Robot]
    ) -> Optional[Robot]:
        if waiting_robot.waiting_since is None:
            return None
        if datetime.now() - waiting_robot.waiting_since < self.DEADLOCK_WAIT_THRESHOLD:
            return None

        next_zone = self._next_desired_zone(waiting_robot)
        if next_zone is None:
            return None

        blockers = self.tm.holder_of(next_zone) - {waiting_robot.id}
        for blocker_id in blockers:
            blocker = all_robots.get(blocker_id)
            if blocker is None or blocker.state != RobotNavState.WAITING:
                continue
            blocker_next = self._next_desired_zone(blocker)
            if blocker_next and waiting_robot.id in self.tm.holder_of(blocker_next):
                return blocker
        return None

    def _next_desired_zone(self, robot: Robot) -> Optional[str]:
        for zid in robot.path:
            if zid not in robot.holdings:
                return zid
        return None

11.5 YieldResolver (Backtrack 방식)

class YieldResolver:
    MAX_RETREAT_DISTANCE_ZONES = 3  # 소형 맵 기준

    def __init__(self, tm: TrafficManager, nav2_client, zones):
        self.tm = tm
        self.nav2 = nav2_client
        self.zones = zones

    def resolve(self, robot_a: Robot, robot_b: Robot) -> None:
        loser, winner = self._pick_loser(robot_a, robot_b)

        # Step 1: backtrack target 탐색
        retreat_zone_id = self._find_backtrack_target(loser, winner)
        if retreat_zone_id is None:
            # Fallback: reverse yield
            retreat_zone_id = self._find_backtrack_target(winner, loser)
            if retreat_zone_id is None:
                loser.state = RobotNavState.ERROR
                winner.state = RobotNavState.ERROR
                return
            loser, winner = winner, loser

        # Step 2: 기존 holdings 해제
        self.tm.release_all(loser.id)
        loser.holdings.clear()

        # Step 3: retreat 경로 예약 (history 역순)
        retreat_path: list[str] = []
        for zid in loser.traversal_history:
            retreat_path.append(zid)
            if zid == retreat_zone_id:
                break
        for zid in retreat_path:
            if not self.tm.try_acquire(loser.id, zid):
                loser.state = RobotNavState.ERROR
                return
            loser.holdings.add(zid)

        # Step 4: 원 목적지 기억
        if loser.original_goal is None:
            last_zone = self.zones[loser.path[-1]]
            loser.original_goal = (
                last_zone.polygon.centroid.x,
                last_zone.polygon.centroid.y,
            )

        # Step 5: 상태 전환
        loser.state = RobotNavState.YIELDING
        loser.yield_target_zone = retreat_zone_id
        loser.yield_for_robot = winner.id

        # Step 6: Nav2 navigate_to_pose (표준 forward navigation)
        retreat_zone = self.zones[retreat_zone_id]
        self.nav2.send_goal(
            loser.id,
            goal_x=retreat_zone.polygon.centroid.x,
            goal_y=retreat_zone.polygon.centroid.y,
        )
        # Nav2가 자동 처리: 제자리 회전 → 전진 → 도착

    def _pick_loser(self, a: Robot, b: Robot) -> tuple[Robot, Robot]:
        # 1순위: priority
        if a.priority != b.priority:
            return (a, b) if a.priority < b.priority else (b, a)
        # 2순위: FIFO
        a_wait = a.waiting_since or datetime.max
        b_wait = b.waiting_since or datetime.max
        if a_wait != b_wait:
            return (a, b) if a_wait > b_wait else (b, a)
        # 3순위: ID 사전순
        return (a, b) if a.id < b.id else (b, a)

    def _find_backtrack_target(self, loser: Robot, winner: Robot) -> Optional[str]:
        winner_future = set(winner.path)
        retreated = 0
        for past_zone_id in loser.traversal_history:
            retreated += 1
            if retreated > self.MAX_RETREAT_DISTANCE_ZONES:
                return None
            if past_zone_id in winner_future:
                continue
            past_zone = self.tm.zones[past_zone_id]
            if not past_zone.can_accept(loser.id):
                continue
            return past_zone_id
        return None

11.6 RobotController (상태 머신)

class RobotController:
    def __init__(self, sliding, detector, yielder, nav2_client, zones):
        self.sliding = sliding
        self.detector = detector
        self.yielder = yielder
        self.nav2 = nav2_client
        self.zones = zones

    def tick(self, robot: Robot, all_robots: dict[str, Robot]) -> None:
        handlers = {
            RobotNavState.IDLE:       self._on_idle,
            RobotNavState.NAVIGATING: self._on_navigating,
            RobotNavState.WAITING:    self._on_waiting,
            RobotNavState.YIELDING:   self._on_yielding,
            RobotNavState.YIELDED:    self._on_yielded,
            RobotNavState.RESUMING:   self._on_resuming,
            RobotNavState.ERROR:      self._on_error,
        }
        handlers[robot.state](robot, all_robots)

    def _on_idle(self, robot, _): pass

    def _on_navigating(self, robot, _):
        if self.sliding.update(robot):
            robot.waiting_since = None
        else:
            robot.state = RobotNavState.WAITING
            robot.waiting_since = datetime.now()
            self.nav2.pause(robot.id)

    def _on_waiting(self, robot, all_robots):
        if self.sliding.update(robot):
            robot.state = RobotNavState.NAVIGATING
            robot.waiting_since = None
            self.nav2.resume(robot.id)
            return
        partner = self.detector.find_deadlock_partner(robot, all_robots)
        if partner is not None:
            self.yielder.resolve(robot, partner)

    def _on_yielding(self, robot, _):
        target = self.zones[robot.yield_target_zone]
        if target.polygon.contains(Point(*robot.position)):
            robot.state = RobotNavState.YIELDED

    def _on_yielded(self, robot, all_robots):
        winner = all_robots.get(robot.yield_for_robot)
        if winner is None or winner.state in (
            RobotNavState.IDLE, RobotNavState.NAVIGATING
        ):
            robot.state = RobotNavState.RESUMING

    def _on_resuming(self, robot, _):
        new_path = self.nav2.plan_path(robot.position, robot.original_goal)
        robot.path = self.nav2.path_to_zones(new_path)
        robot.holdings.clear()
        robot.yield_target_zone = None
        robot.yield_for_robot = None
        robot.state = RobotNavState.NAVIGATING
        self.nav2.send_goal(robot.id, *robot.original_goal)

    def _on_error(self, robot, _):
        # 알람 발송 + 수동 개입 대기
        pass

class FleetCoordinator:
    TICK_HZ = 20  # 소형 맵 빠른 반응

    def __init__(self, zones: dict[str, Zone], nav2_client):
        self.tm = TrafficManager(zones)
        self.sliding = SlidingWindowPlanner(self.tm)
        self.detector = DeadlockDetector(self.tm)
        self.yielder = YieldResolver(self.tm, nav2_client, zones)
        self.controller = RobotController(
            self.sliding, self.detector, self.yielder, nav2_client, zones
        )
        self.robots: dict[str, Robot] = {}

    def register_robot(self, robot: Robot) -> None:
        self.robots[robot.id] = robot

    def dispatch(self, robot_id, goal, priority) -> None:
        robot = self.robots[robot_id]
        path = self.yielder.nav2.plan_path(robot.position, goal)
        robot.path = self.yielder.nav2.path_to_zones(path)
        robot.priority = priority
        robot.original_goal = goal
        robot.state = RobotNavState.NAVIGATING
        robot.holdings.clear()
        robot.traversal_history.clear()

    def tick(self) -> None:
        for robot in self.robots.values():
            self.controller.tick(robot, self.robots)
        self.tm.cleanup_stale()

⑰ [v1.1] compute_path의 한계와 그래프 기반 대안 중요

17.1 발견된 허점

섹션 ①~⑯의 설계에는 간과한 문제가 있습니다. Nav2의 compute_path_to_pose는 우리 TrafficManager의 Zone 예약 상태를 전혀 모릅니다.

Nav2의 global planner(NavFn/SmacPlanner)가 참조하는 것은 static costmap(벽, 기둥 같은 물리 장애물)뿐입니다. 다른 로봇이 Zone을 예약 중이더라도 costmap에는 반영되지 않으므로, Nav2는 그 Zone을 지나는 최단 경로를 아무 문제 없이 반환합니다.

결과: AMR-B가 receive한 "최단 경로"는 AMR-A가 이미 예약한 Zone을 통과할 가능성이 높습니다. 출발 후 WAITING에 빠지거나 deadlock으로 이어집니다. 우회 경로가 맵에 존재하더라도 Nav2는 그것을 알 수 없습니다.

17.2 구체적 시나리오

맵: A ─ z1 ─ z2 ─ z3 ─ z4 ─ z5 ─ B │ │ └── z6 ─ z7 ────┘ ← 우회 경로 (조금 더 길지만 존재) 상황: AMR-1: 현재 A → 목표 B Nav2 compute_path → [z1, z2, z3, z4, z5] (최단) 예약: [z1, z2, z3] AMR-2: 현재 B → 목표 A Nav2 compute_path → [z5, z4, z3, z2, z1] (최단, AMR-1 예약 무시!) ↓ 출발 후 z3에서 WAITING → deadlock → yield → 우회로 [z5, z7, z6, z1]가 있는데 사용되지 않음. 비효율.

17.3 세 가지 해결 방식

방식접근장점단점
A. Costmap Injection 예약된 Zone을 Nav2 costmap에 dynamic obstacle로 주입 Nav2 표준 기능 그대로 사용 로봇별 개별 costmap 필요, 동기화 복잡
B. Replan on WAITING 막힌 후 예약 회피하여 재계산 평시 오버헤드 없음 이미 출발한 후 되돌아감, 비효율
C. Graph-Based Planning 채택 Waypoint 그래프 위에서 직접 Dijkstra, 예약 회피 포함 완벽한 회피, 재계획 저비용 Nav2 기본 planner 우회
채택: 방식 C — 그래프 기반 계획. Dijkstra 실행 시 다른 로봇이 예약한 Zone/Edge를 cost = ∞로 처리하여 자동 우회. 이 선택은 섹션 ⑱의 Waypoint/Edge 추상화 재정립, 섹션 ⑲의 nav2_route Route Server 채택으로 이어집니다.

17.4 개선 효과 예시

방식 C 적용 후 동일 시나리오: AMR-1 dispatch: blocked={} → 경로 [z1, z2, z3, z4, z5] AMR-2 dispatch: blocked={z1, z2, z3} ← AMR-1 예약 → Dijkstra가 [z5, z7, z6, z1] 자동 선택 → 우회로로 바로 출발 결과: 충돌 없음, WAITING 없음, yield 없음, deadlock 사전 예방

18.2 핵심 개념: 예약은 "Edge"에 건다

Waypoint 방식에서 예약의 단위가 달라집니다:

Edge 점유

로봇이 waypoint A에서 waypoint B로 이동 중일 때 해당 edge를 점유. 도착하면 즉시 해제.

충돌 유형: 같은 edge를 반대 방향으로 점유 시도 → 정면 충돌(deadlock)

Waypoint 점유

로봇이 waypoint에서 작업 수행 또는 대기 중일 때 해당 waypoint를 점유. 어떤 edge도 점유하지 않음.

충돌 유형: 같은 waypoint에 capacity 초과 진입 시도

18.6 알고리즘은 그대로 재사용

B + C 하이브리드의 핵심 로직은 데이터 구조만 교체하면 그대로 동작합니다:

  • Dijkstra: Zone polygon 대신 Waypoint 노드 사용, Edge cost로 길이
  • Sliding Window: "앞쪽 N개 edge를 미리 예약"
  • Deadlock 감지: 두 로봇이 서로 다른 방향으로 같은 edge를 원함
  • Backtrack Yield: loser가 직전 waypoint로 1-edge 역방향 이동
  • Nav2 통합: navigate_to_pose(waypoint.position) — 제자리 회전 + 전진
요약: 섹션 ①~⑯의 Polygon Zone 방식은 개념 설계로서 유효하지만, 1m×2m 소형 맵 실제 구현은 본 섹션의 Waypoint+Edge 방식으로 진행합니다.

19.7 예약 상태 → Route Server 브릿지 코드

import rclpy
from rclpy.node import Node
from nav2_route_msgs.msg import DynamicEdges

class ReservationToRouteBridge(Node):
    """
    우리 TrafficManager의 예약 상태를 nav2_route의
    DynamicEdgesScorer에 전달하는 브릿지 노드.
    """

    def __init__(self, traffic_manager):
        super().__init__("reservation_to_route_bridge")
        self.tm = traffic_manager

        # DynamicEdgesScorer가 구독하는 토픽에 차단 edge 발행
        self.pub = self.create_publisher(
            DynamicEdges,
            "/dynamic_edges",
            qos_profile=10,
        )
        self.timer = self.create_timer(0.1, self.publish_blocked)  # 10Hz

    def publish_blocked(self):
        msg = DynamicEdges()
        # 다른 로봇이 사용 중인 edge를 "차단됨" 으로 표시
        for edge_id, edge in self.tm.edges.items():
            if edge.current_users:
                msg.closed_edges.append(edge_id)
        self.pub.publish(msg)

20.4 Fallback 전략 (Route Server 미지원 시)

만약 ROS 2 Jazzy에서 nav2_route 설치가 불가능하거나 버전 호환 문제가 발생하면, 섹션 ⑱의 자체 구현 방식으로 되돌립니다:
  1. Waypoint/Edge 데이터 구조 자체 정의 (약 100줄)
  2. Dijkstra 직접 구현 (약 80줄)
  3. Nav2는 저수준 navigate_to_pose만 사용
  4. 재계획은 RobotController._on_waiting에서 수동 트리거
이 경우에도 전체 아키텍처와 알고리즘은 동일하게 유지됩니다.
