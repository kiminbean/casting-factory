3 계층별 상세 분석

Client 주문 폼 흐름

src/app/customer/page.tsx 에서 5단계 스텝 위저드로 구현

Step이름수집 데이터
1주문자 정보회사명, 담당자, 연락처, 이메일, 주소
2제품 선택제품 ID (D450, D600, D800, GRATING)
3사양 입력직경, 두께, 하중등급, 재질, 수량, 납기, 후처리
4견적 확인단가·합계 계산 확인 (입력 없음)
5주문 완료서버 응답 표시 (입력 없음)
Step 4 → 5 전환 시 handleNext()가 2번의 순차 fetch() 호출로 주문 헤더 + 품목 상세를 백엔드에 전송합니다.

Proxy Next.js Rewrite 설정

next.config.ts 

// next.config.ts
async rewrites() {
  return [
    {
      source: "/api/:path*",
      destination: "http://localhost:8000/api/:path*",
    },
  ];
}
클라이언트의 /api/* 요청을 동일 도메인에서 보내므로 CORS 문제 없이 백엔드로 프록시됩니다. 브라우저 네트워크 탭에서는 localhost:3000/api/orders로 표시됩니다.

Backend FastAPI 처리 흐름

backend/app/routes/orders.py

  1. 요청 수신: Uvicorn이 HTTP POST 수신
  2. Pydantic 검증: OrderCreate 스키마로 자동 타입/필드 검증
  3. 중복 체크: 동일 ID 주문 존재 시 409 Conflict 반환
  4. ORM 매핑: Order(**payload.model_dump())
  5. DB 저장: db.add(order) → db.commit()
  6. 응답 반환: db.refresh(order) 후 201 Created

Database 저장 구조

PostgreSQL 16 + TimescaleDB (개발 시 SQLite 폴백)

테이블용도레코드 생성 시점
orders주문 헤더 (고객·금액·상태)Step 4 → 5 첫 번째 fetch
order_details품목 상세 (제품·사양·가격)Step 4 → 5 두 번째 fetch
orders.id는 클라이언트에서 생성한 ORD-YYYYMMDD-XXXX 형식. order_details.order_id가 FK로 연결됩니다.
