① 개요

Shipment Lane Queue는 출고 작업들의 대기열입니다. 재고에서 제품을 꺼내 배송 처리하는 작업은 서로 독립적이므로 여러 AMR이 동시에 수행할 수 있어 capacity > 1입니다.

1.1 역할

  • Scheduler에서 넘어온 출고 작업을 대기
  • 동시에 여러 개를 active로 승격 가능 (AMR 수만큼)
  • 각 출고는 LIFO/FIFO 재고 정책 적용

② 왜 capacity=N인가

출고 작업은 공유 자원 충돌이 적습니다:

  • 창고 → 출고장 이송: AMR N대가 동시 이동 가능 (Traffic Manager가 경로 조율)
  • 서로 다른 제품을 꺼냄: storage_location이 다르면 경합 없음
  • 공통 병목은 출고장 도크뿐

2.1 Capacity 결정 공식

capacity = min(
    사용 가능 AMR 수,        # 예: 3
    출고장 도크 수,           # 예: 2
    MAX_PARALLEL_SHIPMENTS    # 환경변수 상한, 예: 5
)

본 프로젝트 기본값: capacity = 3 (AMR 3대 기준)

④ 자료구조

@dataclass
class ShipmentLaneQueue:
    capacity: int = 3
    queue: list = field(default_factory=list)       # heapq
    active: dict[int, QueueEntry] = field(default_factory=dict)
    # key=work_order_id, value=QueueEntry

    def is_full(self) -> bool:
        return len(self.active) >= self.capacity

    def slots_available(self) -> int:
        return self.capacity - len(self.active)

⑤ 병렬 활성화 알고리즘

Production과 달리, enqueue 시 또는 complete 시 여러 개를 연속 승격합니다:

def _try_activate_batch(self) -> list[int]:
    """사용 가능한 slot만큼 큐 front에서 승격."""
    activated = []
    while self.slots_available() > 0 and self._heap:
        entry = heappop(self._heap)
        self._active[entry.work_order_id] = entry

        wo = self.db.query(WorkOrder).get(entry.work_order_id)
        wo.work_status = "running"
        wo.actual_start_at = datetime.utcnow()
        activated.append(wo.work_order_id)

        emit_event("shipment.activated", work_order_id=wo.work_order_id)

    self.db.commit()
    return activated
Fairness 주의: 병렬 승격 시에도 우선순위 순서를 지켜야 함. 저우선순위 출고가 먼저 들어왔어도 고우선순위 뒤에 승격되어야 함. heapq가 자동으로 이를 보장.

⑥ LIFO/FIFO 출고 정책

Confluence GUI 페이지에서 언급된 LIFO/FIFO 정책은 큐 자체가 아니라 재고에서 어떤 제품을 꺼낼지의 결정 규칙입니다.

정책의미적합 상황
LIFO (Last In First Out) 가장 최근 입고 제품부터 출고 유통기한 영향 없음, 접근성 우선 (주물처럼 변질 없음)
FIFO (First In First Out) 가장 오래된 입고 제품부터 출고 재고 회전, 변질/부식 우려
본 프로젝트 기본: LIFO. 주물은 변질 우려가 없고, 창고 랙 최상단에서 꺼내는 것이 AMR 동선상 효율적.

6.1 정책 적용 위치

LIFO/FIFO는 Shipment Queue가 아니라 Resource Reservation Manager의 storage_location 선택 로직에서 처리:

def pick_storage_location(product_id, quantity, policy: str):
    if policy == "LIFO":
        order_by = storage_location.c.stored_at.desc()
    else:  # FIFO
        order_by = storage_location.c.stored_at.asc()

    candidates = db.query(StorageLocation).filter_by(
        product_id=product_id,
        status="occupied",
    ).order_by(order_by).limit(quantity).all()
    return candidates

⑦ 구현 스케치

class ShipmentLaneQueue:
    CAPACITY = 3

    def __init__(self, db_session):
        self.db = db_session
        self._heap: list[QueueEntry] = []
        self._active: dict[int, QueueEntry] = {}
        self._lock = threading.RLock()

    def enqueue(self, work_order) -> None:
        with self._lock:
            entry = QueueEntry(
                sort_key=(
                    -work_order.priority_score,
                    work_order.released_at,
                    work_order.work_order_id,
                ),
                work_order_id=work_order.work_order_id,
                priority_score=work_order.priority_score,
                released_at=work_order.released_at,
                lane="shipment",
            )
            heappush(self._heap, entry)

            wo = work_order
            wo.work_status = "queued"
            self.db.commit()

            self._try_activate_batch()

    def complete(self, work_order_id: int, success: bool) -> None:
        with self._lock:
            if work_order_id not in self._active:
                return
            self._active.pop(work_order_id)

            wo = self.db.query(WorkOrder).get(work_order_id)
            wo.work_status = "completed" if success else "failed"
            wo.actual_end_at = datetime.utcnow()
            self.db.commit()

            self._try_activate_batch()  # 빈 slot 채우기

    def active_count(self) -> int:
        return len(self._active)

    def update_capacity(self, new_cap: int) -> None:
        """AMR 고장 / 복구 시 capacity 동적 조정."""
        with self._lock:
            self.capacity = new_cap
            self._try_activate_batch()  # 증가 시 즉시 승격
