① 개요

Global Task Allocator는 Ready Task Pool의 task를 실제 로봇/설비에 배정하는 의사결정 엔진입니다. 생산 lane과 출고 lane의 task를 함께 보고 최적 배정을 수행합니다.

1.1 핵심 책임

  • 사용 가능한 로봇/설비 목록 유지
  • 각 task에 대해 후보 executor 평가
  • 점수가 가장 높은 (robot, task) 쌍 선택
  • 자원 예약(Reservation Manager)과 경로 예약(Traffic Manager) 검증

1.2 범위 밖

  • 실제 명령 송출 — Robot/Cell Executors
  • Station 점유 관리 — Reservation Manager
  • AMR 경로 충돌 — Traffic Manager

② 매칭 문제 정의

이 문제는 Assignment Problem의 간소화 버전입니다:

Task Pool 로봇 Pool ┌─ T1 (AMR) ┌─ AMR-01 (idle, 배터리 78%, 위치 A) ├─ T2 (AMR) ├─ AMR-02 (idle, 배터리 45%, 위치 B) ├─ T3 (Cobot pour) ├─ AMR-03 (working) ├─ T4 (Cobot demold) ├─ Cobot-01 (idle, pouring-capable) └─ T5 (vision) └─ Cobot-02 (idle, demolding-capable) → 최적 매칭: T1 → AMR-01 (가깝고 배터리 충분) T2 → AMR-02 (배터리 45%, 여유 있음) T3 → Cobot-01 T4 → Cobot-02 T5 → Jetson (별도 큐)

③ 평가 함수

(robot, task) 쌍의 적합도를 0~100 점수로 계산:

def score(robot: Robot, task: Task) -> float:
    if not robot.can_handle(task):
        return -math.inf  # 자격 미달

    s = 0.0

    # 1) 거리 (40점) — 가까울수록 높음
    distance = euclidean(robot.position, task.origin)
    s += max(0, 40 - distance * 10)  # 4m 이내는 만점

    # 2) Task 우선순위 (30점)
    s += (task.priority_score / 100) * 30

    # 3) 배터리 여유 (20점) — task 수행 후 20% 이상 남을 것
    estimated_consumption = estimate_battery(task)
    after_battery = robot.battery - estimated_consumption
    if after_battery >= 20:
        s += 20
    elif after_battery >= 10:
        s += 10
    else:
        s += 0  # 위험

    # 4) Capability 특화 (10점) — 정확히 요구 스킬 가진 로봇 우대
    if set(task.required_capability) == set(robot.capabilities):
        s += 10
    elif set(task.required_capability).issubset(robot.capabilities):
        s += 5

    return s
가중치 조정: 위 값들은 기본값. 현장 특성에 따라 environment variable로 조정 가능. 예: ALLOCATOR_DISTANCE_WEIGHT=30, ALLOCATOR_BATTERY_WEIGHT=25

④ 매칭 알고리즘

4.1 단순 Greedy (현재 구현)

def allocate_tick():
    idle_robots = get_idle_robots()
    ready_tasks = ready_pool.size_by_capability()

    if not idle_robots or not ready_tasks:
        return

    # 각 idle robot에 대해 최적 task 탐색
    for robot in idle_robots:
        best_task = None
        best_score = -math.inf

        for cap in robot.capabilities:
            task = ready_pool.peek_for(cap)
            if task and score(robot, task) > best_score:
                best_score = score(robot, task)
                best_task = task

        if best_task:
            claimed = ready_pool.claim(best_task.task_id)
            if claimed:
                assign_robot_to_task(robot, claimed)

4.2 Hungarian Algorithm (향후 개선)

N개 robot, M개 task가 있을 때 이론적으로 최적 매칭 (O(N³)). 3대 AMR 규모에서는 Greedy로 충분. 7대+ 확장 시 검토.

4.3 Reservation 검증

매칭 결정 후, 실제 assign 전에 자원/경로 확보 확인:

def assign_robot_to_task(robot, task):
    # Step 1: Resource Reservation 확인
    if task.requires_station:
        if not resource_mgr.reserve(task.station_id, robot.id):
            return False  # 나중에 재시도

    # Step 2: Traffic Manager 경로 예약
    if robot.type == "amr":
        path = traffic_mgr.compute_safe_path(robot.position, task.destination)
        if path is None:
            resource_mgr.release(task.station_id, robot.id)
            return False

    # Step 3: 실제 assign
    task.state = TaskState.ASSIGNED
    task.assigned_to = robot.id
    robot.current_task_id = task.task_id
    db.commit()

    # Step 4: Executor에게 명령 송출
    executor.dispatch(robot, task)
    return True

⑤ 배터리 & 충전 관리

AMR의 배터리는 매우 중요한 제약:

배터리 레벨상태Allocator 동작
> 60%정상모든 task 배정 가능
30~60%주의긴 경로 task 피함
< 30%충전 필요자동으로 "충전소 복귀" task 삽입 (최고 우선순위)
< 15%위험모든 task 거부, 즉시 충전

⑥ 구현 스케치

class GlobalTaskAllocator:
    TICK_HZ = 2  # 500ms 주기

    def __init__(self, ready_pool, resource_mgr, traffic_mgr, fleet):
        self.ready_pool = ready_pool
        self.resource_mgr = resource_mgr
        self.traffic_mgr = traffic_mgr
        self.fleet = fleet  # 모든 로봇/설비

    def tick(self):
        # 1) 배터리 체크
        self._check_battery()

        # 2) idle 로봇 나열
        idle = [r for r in self.fleet if r.state == "idle"]
        if not idle:
            return

        # 3) 각 로봇에 최적 task 탐색
        assignments = []
        claimed_ids = set()
        for robot in sorted(idle, key=lambda r: -r.priority_bonus):
            best = self._find_best_task(robot, exclude=claimed_ids)
            if best:
                assignments.append((robot, best))
                claimed_ids.add(best.task_id)

        # 4) 실제 assign
        for robot, task in assignments:
            self._try_assign(robot, task)

    def _find_best_task(self, robot, exclude):
        best_task = None
        best_score = -math.inf
        for cap in robot.capabilities:
            task = self.ready_pool.peek_for(cap)
            if task and task.task_id not in exclude:
                s = self._score(robot, task)
                if s > best_score:
                    best_score = s
                    best_task = task
        return best_task

    def _try_assign(self, robot, task):
        # reservation + traffic 검증
        if not self._reserve_resources(robot, task):
            return False
        claimed = self.ready_pool.claim(task.task_id)
        if claimed is None:
            self._release_resources(robot, task)
            return False
        claimed.state = TaskState.ASSIGNED
        claimed.assigned_to = robot.id
        emit_event("task.assigned", task_id=claimed.task_id, robot_id=robot.id)
        return True

⑦ 실패 처리

실패 원인복구 동작
Reservation 실패task를 Pool에 반환, 다음 tick에서 재시도
Traffic path 없음10초 후 재시도, 반복 실패 시 Monitor에 escalate
Executor 실행 실패task.state = FAILED, Workflow에 알림 (재시도 판단)
Robot 통신 두절robot.state = unavailable, task를 Pool 반환
