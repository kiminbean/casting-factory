③ 계층별 책임 요약

계층컴포넌트핵심 책임주요 출력
Presentation Office UI 고객 주문 접수, 사무실 승인 워크플로우 POST /api/orders
Factory Manager UI 현장 관리자 대시보드, 생산 개시 트리거 PATCH release
API/DB Order API / DB 상태 머신 영속화, REST API work_order CRUD
Planning Work Order Scheduler 우선순위 계산, lane 배정 결정 lane별 active order
Queue Production Lane Queue 생산 1개씩 순차 진행 next production order
Shipment Lane Queue 출고 N개 병렬 진행 active shipments
Workflow Workflow / Task Manager order → task DAG, 선행조건 해결 task 생성/상태전이
Ready Task Pool 실행 가능 task 집합 관리 pickable tasks
Allocation Global Task Allocator task ↔ 로봇/설비 매칭 task assignment
Resource Resource Reservation Manager station/buffer/dock 점유 reservation grant/deny
Traffic Manager AMR 경로 충돌 방지 zone reservation
Execution Robot / Cell Executors 실제 명령 송출 (ROS2 action, MQTT) execution result
Monitoring Execution Monitor 결과 수집, 예외 처리, 재시도 DB 갱신, UI 알림

④ 읽는 순서 권장

처음 이 아키텍처를 학습하는 경우:

  1. 02 Order API / DB — 전체 상태 모델 이해의 출발점
  2. 01 Office UI + 03 Factory Manager UI — 사람이 보는 면
  3. 04 Work Order Scheduler — 어떤 order가 언제 실행되는지 결정
  4. 05, 06 Lane Queues — 왜 lane이 나뉘는지 이해
  5. 07 Workflow / Task Manager — 고수준 order가 어떻게 세부 task로 쪼개지는지
  6. 08 Ready Task Pool — dependency 해결과 실행 대기열
  7. 09 Global Task Allocator — task ↔ 로봇 매칭 의사결정
  8. 10 Resource Reservation + Traffic Manager — 물리적 충돌 방지
  9. 11 Robot / Cell Executors — 실제 장비 제어
  10. 12 Execution Monitor — 결과 루프 닫기

  • NFR 수치 명시: 응답 시간, 처리량 등 정량 기준
