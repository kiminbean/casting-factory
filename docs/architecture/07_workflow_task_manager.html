① 개요

Workflow / Task Manager는 고수준의 work_order를 원자적인 task들로 쪼개고, 이들 사이의 선행 관계(dependency)를 관리합니다. 생산 1건은 20~30개의 task로 분해됩니다.

1.1 역할

  • Template 매칭: work_order 타입에 따라 task 템플릿 선택
  • DAG 생성: task 노드 + 선행 관계 엣지
  • Dependency 해결: 선행 task 완료 감지 → 다음 task ready 전환
  • Ready 이벤트 발행: Ready Task Pool에 삽입

1.2 역할이 아닌 것

  • 로봇 선택 — Allocator
  • 경로 계획 — Traffic Manager
  • 자원 예약 — Reservation Manager

② Work Order → Task 분해

work_order의 제품·공정 특성에 따라 사전 정의된 템플릿으로 분해:

Work Order 타입Task 개수주요 Task
주조 생산 (신규) ~25 원재료 준비/용해/조형/주탕/냉각/탈형/이송/검사/분류/적재
출고 (재고에서) ~6 위치 조회/피킹/이송/도크 도착/출고 확인/재고 차감
재배치 (Rebalancing) ~4 소스 픽업/이송/목적지 적재/완료 기록

③ DAG 모델

from dataclasses import dataclass, field
from enum import Enum

class TaskType(Enum):
    MATERIAL_INPUT = "material_input"
    MELTING = "melting"
    MOLDING = "molding"
    POURING = "pouring"
    COOLING = "cooling"
    DEMOLDING = "demolding"
    TRANSPORT = "transport"
    INSPECTION = "inspection"
    SORTING = "sorting"
    STORAGE = "storage"
    PICKING = "picking"
    SHIPMENT = "shipment"

class TaskState(Enum):
    PENDING = "pending"      # 선행 미완료
    READY = "ready"          # Pool에 삽입 가능
    ASSIGNED = "assigned"    # Allocator가 로봇 배정
    RUNNING = "running"      # 실행 중
    COMPLETED = "completed"
    FAILED = "failed"

@dataclass
class Task:
    task_id: str
    work_order_id: int
    type: TaskType
    state: TaskState = TaskState.PENDING
    required_capability: list[str] = field(default_factory=list)
    # 예: ["amr"], ["manipulator", "pouring"]
    depends_on: list[str] = field(default_factory=list)  # task_ids
    payload: dict = field(default_factory=dict)
    assigned_to: str | None = None
    started_at: datetime | None = None
    ended_at: datetime | None = None

④ Task 상태 머신

[Workflow 생성] │ ▼ PENDING ─────► READY ─────► ASSIGNED ─────► RUNNING ─────► COMPLETED │ │ │ │ │ │ │ └──────► FAILED │ │ │ │ │ └── (재배정 필요 시) ──► READY │ │ │ └── (allocator가 pickup) ──► ASSIGNED │ └── (depends_on 모두 COMPLETED) ──► READY

def on_task_completed(completed_task: Task):
    """한 task가 완료되면 그것에 의존하던 task들을 READY로 승격."""
    dependents = db.query(Task).filter(
        Task.depends_on.contains([completed_task.task_id])
    ).all()

    for dep in dependents:
        if all_deps_completed(dep):
            dep.state = TaskState.READY

[원재료 준비] │ ▼ [용해 시작] │ ▼ [용해 완료 감지] │ ┌───────┴───────┐ (병렬) ▼ ▼ [조형 시작] [주탕 위치 이동] │ │ ▼ │ [조형 완료] │ │ │ └───────┬───────┘ ▼ [주탕] │ ▼ [냉각 모니터링] │ ▼ [냉각 완료 판정] │ ▼ [탈형] │ ▼ [AMR 이송: 냉각→후처리] │ ▼ [후처리 (수작업)] │ ▼ [AMR 이송: 후처리→검사] │ ▼ [컨베이어 구동] │ ▼ [비전 검사] / \ ▼ ▼ [양품 분류] [불량 분류] │ │ ▼ ▼ [AMR 양품 적재] [AMR 불량 폐기]

병렬 구간: 조형과 주탕 위치 이동은 동시 진행 가능. Task Manager는 이런 병렬 구간을 DAG로 표현.

⑦ 구현 스케치

class WorkflowTaskManager:
    def __init__(self, db, ready_pool):
        self.db = db
        self.pool = ready_pool

    def decompose(self, work_order) -> list[Task]:
        """Work order를 task DAG로 분해."""
        template = self._select_template(work_order)
        tasks = []

        for task_spec in template.tasks:
            task = Task(
                task_id=f"WO{work_order.work_order_id}-T{task_spec.seq}",
                work_order_id=work_order.work_order_id,
                type=task_spec.type,
                required_capability=task_spec.capability,
                depends_on=[
                    f"WO{work_order.work_order_id}-T{dep}"
                    for dep in task_spec.depends_on
                ],
                payload=task_spec.payload_for(work_order),
            )
            tasks.append(task)
            self.db.add(task)

        self.db.commit()

        # dependency가 없는 task는 즉시 READY
        for task in tasks:
            if not task.depends_on:
                self._mark_ready(task)

        return tasks

    def on_task_completed(self, task_id: str, success: bool):
        task = self.db.query(Task).get(task_id)
        task.state = TaskState.COMPLETED if success else TaskState.FAILED
        task.ended_at = datetime.utcnow()
        self.db.commit()

        if not success:
            self._handle_task_failure(task)
            return

        # dependency 해결
        dependents = self.db.query(Task).filter(
            Task.depends_on.any(task_id)
        ).all()
        for dep in dependents:
            if self._all_deps_done(dep):
                self._mark_ready(dep)

        # 모든 task 완료 시 work_order 완료
        if self._all_wo_tasks_done(task.work_order_id):
            self._complete_work_order(task.work_order_id)

    def _mark_ready(self, task: Task):
        task.state = TaskState.READY
        self.db.commit()
        self.pool.insert(task)
        emit_event("task.ready", task_id=task.task_id)
