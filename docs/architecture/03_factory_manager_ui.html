① 개요

Factory Manager UI는 공장 현장의 관리자(operator/supervisor)가 사용하는 인터페이스입니다. 사무실에서 승인된 주문을 받아 실제로 생산할 수 있는 상태인지 현장에서 최종 판단하고 release 신호를 보내는 것이 핵심 역할입니다.

④ Release 워크플로우

사용자 아키텍처 다이어그램에서 명시된 "생산 시작" 버튼의 실제 동작:

  1. 주문 선택: 승인 상태(approved) 다중 체크박스
  2. 우선순위 계산: POST /api/production/schedule/calculate (dry-run, DB 상태 변경 없음)
  3. 결과 검토: 7요소 점수 바 + 추천 사유 + 착수 불가 경고
  4. 수동 조정 (선택): ↑/↓ 버튼으로 순서 변경, 사유 입력 모달 필수
  5. 확인 모달: "정말 생산 시작하시겠습니까?"
  6. 생산 시작: POST /api/production/schedule/start
    • DB: customer_order.status: approved → in_production
    • DB: work_order 생성 + work_status = released
    • WebSocket broadcast로 대시보드 즉시 갱신
핵심: release 버튼은 단순 상태 플립이 아니라 현장 "확약"입니다. 관리자가 자재/패턴/로봇 가용성을 눈으로 확인한 후 누르는 행동 — 잘못 누르면 실제 생산 라인이 작동함.

⑤ PyQt5 Factory PC

현장에 설치된 대형 모니터 전용 데스크탑 앱. MQTT 실시간 구독으로 설비/AMR 상태를 즉시 반영.

  • Dashboard: KPI 요약
  • Map: 공장 레이아웃 + AMR 위치
  • Production: 공정 흐름 카드
  • Logistics: 물류/이송 상태
  • Quality: 비전 검사 피드

⑥ 구현 스케치

6.1 생산 시작 버튼 (Next.js)

async function startProduction(selectedOrders: Order[], lane: "production") {
  // Step 1: dry-run 우선순위 계산
  const calcRes = await fetch("/api/production/schedule/calculate", {
    method: "POST",
    body: JSON.stringify({ order_ids: selectedOrders.map(o => o.id) }),
  });
  const { ranked } = await calcRes.json();

  // Step 2: 사용자에게 결과 표시 + 수동 조정 UI
  const finalOrder = await showPriorityModal(ranked);
  if (!finalOrder) return;

  // Step 3: 확인 모달
  if (!await confirm("정말 생산을 시작하시겠습니까?")) return;

  // Step 4: 실제 release (상태 전이)
  const res = await fetch("/api/production/schedule/start", {
    method: "POST",
    body: JSON.stringify({
      order_ids: finalOrder.map(o => o.id),
      lane,
      priority_changes: finalOrder.map((o, i) => ({
        order_id: o.id,
        manual_rank: i + 1,
        reason: o.change_reason ?? null,
      })),
    }),
  });
  if (res.ok) {
    toast("생산이 시작되었습니다");
    router.push("/production");  // 모니터링 화면으로
  }
}
