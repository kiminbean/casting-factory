① 개요

Ready Task Pool은 당장 실행 가능한 task들의 집합입니다. Workflow Task Manager가 task를 READY 상태로 승격시키면 이 Pool에 삽입되고, Global Task Allocator가 로봇을 배정할 때 여기서 꺼냅니다.

1.1 특성

  • 다수의 work_order task가 섞여 있음 — 생산 task + 출고 task + 재배치 task
  • 모든 dependency 해결됨 — 즉시 실행 가능
  • Capability별 분류 — AMR task, Manipulator task 등
  • 우선순위 유지 — work_order의 priority_score 상속

③ 자료구조 (다중 인덱스)

Allocator는 "AMR에게 줄 수 있는 task 중 가장 급한 것" 같은 쿼리를 자주 합니다. 따라서 Pool은 capability별 heap을 여러 개 유지합니다:

@dataclass
class ReadyTaskPool:
    # capability → heap[(sort_key, task_id)]
    _heaps: dict[str, list] = field(default_factory=dict)
    # task_id → Task (빠른 조회)
    _index: dict[str, Task] = field(default_factory=dict)
    _lock: threading.RLock = field(default_factory=threading.RLock)

    def insert(self, task: Task):
        with self._lock:
            sort_key = (
                -task.priority_score,
                task.ready_at,     # READY 전환 시각 (FIFO tiebreak)
                task.task_id,
            )
            # 필요한 capability 각각에 추가
            for cap in task.required_capability:
                heap = self._heaps.setdefault(cap, [])
                heappush(heap, (sort_key, task.task_id))
            self._index[task.task_id] = task

    def peek_for(self, capability: str) -> Task | None:
        with self._lock:
            heap = self._heaps.get(capability)
            while heap:
                _, tid = heap[0]
                if tid in self._index:
                    return self._index[tid]
                heappop(heap)  # stale entry 제거
            return None

    def claim(self, task_id: str) -> Task | None:
        """Allocator가 task 소유권을 가져감. Pool에서 제거."""
        with self._lock:
            task = self._index.pop(task_id, None)
            # heap entry는 lazy 삭제 (peek_for에서 처리)
            return task

⑤ Allocator 연동

Allocator가 task를 가져가는 전형적인 패턴:

# Global Task Allocator 측
def allocate_tick():
    for robot in idle_robots:
        for cap in robot.capabilities:
            task = ready_pool.peek_for(cap)
            if task and can_assign(robot, task):
                claimed = ready_pool.claim(task.task_id)
                assign_to_robot(claimed, robot)
                break  # 다음 robot으로

5.1 동시 claim 경쟁

두 Allocator 인스턴스(또는 스레드)가 같은 task를 동시에 claim하려 하면? → _lock으로 직렬화. claim()은 원자적이며, 첫 번째가 pop()한 후 두 번째는 None 받음.

⑥ Starvation 방지

고우선순위 task가 계속 들어오면 저우선순위 task가 영원히 대기할 수 있음. 해결책:

6.1 Aging 알고리즘

def apply_aging(self, interval_sec: int = 60):
    """주기적으로 대기 시간 기반으로 우선순위 가점."""
    now = datetime.utcnow()
    for task in self._index.values():
        wait_time = (now - task.ready_at).total_seconds()
        aging_bonus = int(wait_time / interval_sec)  # 분당 1점
        task.effective_priority = task.priority_score + aging_bonus
    self._reheapify()

6.2 최대 대기 시간 경보

task가 5분 이상 READY 상태인 경우 Execution Monitor에 경보:

def check_stale_tasks(self):
    threshold = timedelta(minutes=5)
    now = datetime.utcnow()
    for task in self._index.values():
        if now - task.ready_at > threshold:
            emit_alert("task_starvation", task_id=task.task_id)

⑦ 구현 스케치

class ReadyTaskPool:
    def __init__(self):
        self._heaps: dict[str, list[tuple]] = {}
        self._index: dict[str, Task] = {}
        self._lock = threading.RLock()

    def insert(self, task: Task) -> None:
        task.state = TaskState.READY
        task.ready_at = datetime.utcnow()
        with self._lock:
            sort_key = (
                -task.priority_score,
                task.ready_at,
                task.task_id,
            )
            for cap in task.required_capability:
                heap = self._heaps.setdefault(cap, [])
                heappush(heap, (sort_key, task.task_id))
            self._index[task.task_id] = task

    def peek_for(self, capability: str) -> Task | None:
        with self._lock:
            heap = self._heaps.get(capability, [])
            while heap:
                _, tid = heap[0]
                if tid in self._index:
                    return self._index[tid]
                heappop(heap)  # stale cleanup
            return None

    def claim(self, task_id: str) -> Task | None:
        with self._lock:
            return self._index.pop(task_id, None)

    def size(self) -> int:
        with self._lock:
            return len(self._index)

    def size_by_capability(self) -> dict[str, int]:
        with self._lock:
            counts = {}
            for cap, heap in self._heaps.items():
                counts[cap] = sum(
                    1 for _, tid in heap if tid in self._index
                )
            return counts
