① 개요

Resource Reservation Manager는 공장 내 정적 자원(station / buffer / dock / charging station)의 점유 상태를 관리합니다. Traffic Manager가 "이동 경로"를 관리한다면 이 매니저는 "머무르는 공간"을 관리합니다.

1.1 대표 자원

  • Station: 용해로 앞, 주탕 위치, 검사대, 후처리 스테이션
  • Buffer: 공정 간 임시 적재 버퍼 (냉각 대기 영역)
  • Dock: 출고장 도크, AMR 피킹 위치
  • Charging Station: AMR 충전소

③ 관리 대상 자원

@dataclass
class Resource:
    id: str
    type: ResourceType  # STATION/BUFFER/DOCK/CHARGING
    position: tuple[float, float]
    capacity: int = 1  # 동시 점유 가능 수
    current_holders: set[str] = field(default_factory=set)
    wait_queue: deque = field(default_factory=deque)
    max_hold_sec: int = 300  # 5분 초과 시 stale 경고

class ResourceType(Enum):
    STATION = "station"
    BUFFER = "buffer"
    DOCK = "dock"
    CHARGING = "charging"

④ 예약 라이프사이클

[Allocator 요청] │ ▼ ┌─ try_reserve(resource_id, holder_id) ────┐ │ │ │ 1. lock 획득 │ │ 2. current_holders < capacity? │ │ YES → add holder, granted │ │ NO → wait_queue 등록, pending │ │ 3. lock 해제 │ └─────────────┬─────────────────────────────┘ │ granted / pending │ ▼ [작업 수행] │ ▼ release(resource_id, holder_id) │ ▼ wait_queue 선두 → grant

4.1 Stale 예약 처리

max_hold_sec 초과 시 관리자 경보 + 강제 해제 옵션:

def cleanup_stale():
    now = datetime.utcnow()
    for res in resources.values():
        for holder, reserved_at in list(res.reserved_at.items()):
            if now - reserved_at > timedelta(seconds=res.max_hold_sec):
                emit_alert("stale_reservation",
                           resource=res.id, holder=holder)
                # 자동 해제는 설정에 따라
                if AUTO_RELEASE_STALE:
                    release(res.id, holder)

⑤ Deadlock 방지

한 task가 여러 자원을 요구하면 데드락 가능 (A→B→A 순환). 해결:

5.1 Resource Ordering (전역 순서)

모든 자원에 고정 순서를 부여하고, 항상 작은 번호부터 획득:

RESOURCE_ORDER = {
    "STN_MELTING": 1,
    "STN_POURING": 2,
    "BUF_COOLING": 3,
    "STN_INSPECT": 4,
    "DOCK_OUT_1": 5,
}

def reserve_multiple(resource_ids: list[str], holder: str):
    # 번호 오름차순 정렬 (데드락 방지)
    sorted_ids = sorted(resource_ids, key=lambda r: RESOURCE_ORDER[r])
    acquired = []
    try:
        for rid in sorted_ids:
            if not try_reserve(rid, holder):
                raise ReservationFailure(rid)
            acquired.append(rid)
        return acquired
    except ReservationFailure:
        # 역순으로 rollback
        for rid in reversed(acquired):
            release(rid, holder)
        raise

5.2 All-or-nothing 방식

일부만 잡고 기다리다가 상대방이 내 것을 원하는 circular wait 방지. 반드시 전부 성공 or 전부 rollback.

⑥ 구현 스케치

class ResourceReservationManager:
    def __init__(self, resources_yaml):
        self.resources: dict[str, Resource] = load_resources(resources_yaml)
        self._lock = threading.RLock()

    def try_reserve(self, rid: str, holder: str) -> bool:
        with self._lock:
            res = self.resources[rid]

            # 이미 보유 (idempotent)
            if holder in res.current_holders:
                return True

            # 용량 여유
            if len(res.current_holders) < res.capacity:
                res.current_holders.add(holder)
                res.reserved_at[holder] = datetime.utcnow()
                return True

            # 대기열 등록 (비차단)
            res.wait_queue.append(holder)
            return False

    def release(self, rid: str, holder: str) -> None:
        with self._lock:
            res = self.resources[rid]
            res.current_holders.discard(holder)
            res.reserved_at.pop(holder, None)

            # 대기열 선두 grant
            while res.wait_queue and len(res.current_holders) < res.capacity:
                next_holder = res.wait_queue.popleft()
                res.current_holders.add(next_holder)
                res.reserved_at[next_holder] = datetime.utcnow()
                emit_event("resource.granted",
                           resource=rid, holder=next_holder)

    def reserve_multiple(self, rids: list[str], holder: str) -> bool:
        """All-or-nothing 다중 예약."""
        sorted_ids = sorted(rids, key=lambda r: RESOURCE_ORDER[r])
        acquired = []
        for rid in sorted_ids:
            if self.try_reserve(rid, holder):
                acquired.append(rid)
            else:
                # rollback
                for r in reversed(acquired):
                    self.release(r, holder)
                return False
        return True

    def status(self, rid: str) -> dict:
        res = self.resources[rid]
        return {
            "id": rid,
            "type": res.type.value,
            "holders": list(res.current_holders),
            "wait_queue": list(res.wait_queue),
            "capacity": res.capacity,
        }
