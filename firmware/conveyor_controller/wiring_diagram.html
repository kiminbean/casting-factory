★ Critical Protocol Discovery (2026-04-06):
This TOF250 module (AliExpress "OvercomeZeus" brand, listing 1005010253988126) does NOT use I2C or binary UART framing despite what most online docs suggest.
Actual protocol: ASCII text at 9600 baud, format "NNN\r\n" (e.g., "18\r\n" = 18mm).
Only the White wire carries data — Yellow, Blue, Green are not connected on the PCB (5 pads total).
The firmware uses String.toInt() parsing with \r\n line terminators, not binary frame parsing.
Conveyor State Machine (firmware v3.0.0):
IDLE → (TOF1 detects object 5-80mm) → RUNNING (motor ON) → (TOF2 detects) → STOPPED (motor OFF, 5s wait for camera inspection) → POST_RUN (motor ON 2s) → CLEARING (motor stays ON until TOF2 clears, 5s safety timeout) → IDLE

Debounce: 500ms on both sensors (must stay detected continuously).
Detection range: Fixed 5-80mm (anything closer or farther = ignored).
Serial Commands (115200 baud via USB):
start / stop — force motor control
reset — return to IDLE, clear cycle count
sim_entry / sim_exit — simulate sensor detection (for testing without physical objects)
status — request immediate status JSON
Verified Working: 2026-04-06. Real sensor test completed 2 successful cycles with motor rotation confirmed. TOF1 readings: 190mm idle → 5mm with object. TOF2 readings: 208mm idle → 6mm with object. State transitions all correct. Debounce working as expected.
