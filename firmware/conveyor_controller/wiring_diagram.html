★ Critical Protocol Discovery (2026-04-06):
This TOF250 module (AliExpress "OvercomeZeus" brand, listing 1005010253988126) does NOT use I2C or binary UART framing despite what most online docs suggest.
Actual protocol: ASCII text at 9600 baud, format "NNN\r\n" (e.g., "18\r\n" = 18mm).
Only the White wire carries data — Yellow, Blue, Green are not connected on the PCB (5 pads total).
The firmware uses String.toInt() parsing with \r\n line terminators, not binary frame parsing.
Conveyor State Machine (firmware v5.0 · conveyor_v5_serial 1.3.0):
IDLE → (TOF1 detects 1-30mm) → RUNNING (motor ON, MIN_RUN 1s) → (TOF2 detects) → STOPPED (motor OFF, wait indefinitely for Jetson RUN/camera_ok/inspect_done) → POST_RUN (motor ON for POST_RUN_MS=4s) → CLEARING (until TOF2 clears, 5s safety timeout) → IDLE

Debounce: 500ms on both TOF sensors · Detection range: 1-30mm.
RFID polling: RC522 healthcheck every 2s, tag hold 700ms, reinit on VersionReg = 0x00 / 0xFF.
★ RFID-RC522 Integration (v5.0, 2026-04-17):
Fixed VSPI wiring (auto-scan removed). Init order: SPI.begin(18,19,23,5) → PCD_Init(5,22) → read VersionReg.
Serial tokens unchanged (BOOT/STOPPED/STARTED/DONE). RFID events publish as JSON: {"event":"rfid_tag","uid":"...","type":"..."} and {"event":"rfid_clear","uid":"..."} after RFID_TAG_HOLD_MS.
Manual commands (USB Serial): RFID_SCAN — force reader reinit; STATUS — include RFID block (ready, spi, ss, rst, uid, type, version).
Two Separate Serial Channels:

① USB Serial (ESP32 ↔ PC): 115200 baud
사용자 명령 및 JSON 로그 통신 (Arduino IDE Serial Monitor)
Commands: start / stop / reset / sim_entry / sim_exit / status

② TOF250 UART (Sensor → ESP32): 9600 baud
센서에서 ESP32로 거리값 전송 (ASCII 형식: "NNN\r\n")
TOF1 → GPIO 16, TOF2 → GPIO 17 (White wire only)

③ RC522 SPI (ESP32 ↔ RFID, VSPI bus)
SS=GP5, SCK=GP18, MOSI=GP23, MISO=GP19, RST=GP22 · 3.3 V power · IRQ NC.
Serial 115200 채널과 충돌하지 않음 (SPI 버스는 별도).
